Los Tres Tipos de Error

¿Por qué mi código no funciona?

Los Tres Tipos de Error

Error de Sintaxis

Escribiste algo que Python no entiende. El programa no arranca.

Error de Ejecución (Runtime)

La sintaxis está bien, pero al correr pides algo imposible.

Error Lógico

El programa corre sin errores, pero el resultado está mal. El más difícil.

Ejemplo de cada uno: 
# Sintaxis — falta los dos puntos
def ingreso(cajas, precio)
    return cajas * precio
# Runtime — división por cero
def promedio(lista):
    return sum(lista) / len(lista)
promedio([])  # ZeroDivisionError
# Lógico — suma en vez de multiplicar
def ingreso(cajas, precio):
    return cajas + precio  # debería ser *
print(ingreso(30, 12.50))  # 42.5 ¡mal!

try / except

Código que no explota

Flujo de try / except

try — intenta ejecutar
¿Hubo error?
except — si falló
else — si NO falló
finally — SIEMPRE

¿Cómo funciona?

Python intenta ejecutar el bloque try. Si algo falla, salta al except correspondiente. Si todo sale bien, ejecuta else. finally se ejecuta siempre, haya error o no.

def leer_precio(texto): try: precio = float(texto) except ValueError: print(f"'{texto}' no es número") return None else: print(f"Precio válido: {precio}") return precio finally: print("--- Proceso terminado ---") leer_precio("12.50") print() leer_precio("abc")

try/except — La Red de Seguridad

Sin protección — explota
def promedio(lista): return sum(lista) / len(lista) print(promedio([])) print("Esto nunca se ejecuta")
Con try/except — controlado
def promedio(lista): try: return sum(lista) / len(lista) except ZeroDivisionError: return 0 print(promedio([])) print("El programa sigue")
  • try: "intenta ejecutar este código".
  • except ZeroDivisionError: "si falla con ESE error, haz esto".
  • El programa no se detiene. Sigue ejecutando lo que viene después.

Atrapar Errores Específicos

Siempre nombren el error. Un except genérico esconde bugs:

Genérico — esconde bugs
try: resultado = 100 / 0 except: print("Algo falló") resultado = 0 print("Resultado:", resultado)
Específico — claro y seguro
try: resultado = 100 / 0 except ZeroDivisionError: print("Error: división por cero") resultado = 0 except KeyError as e: print(f"Clave no encontrada: {e}") print("Resultado:", resultado)

Regla de oro

Cada except debe nombrar el error que espera. Si no saben cuál va a pasar, dejen que explote — así aprenden qué es.

Tres Formas de Importar

# 1. Importar el módulo completo import math print("Raíz de 144:", math.sqrt(144)) print("Pi:", math.pi) # 2. Importar solo lo que necesitas from math import factorial print("5! =", factorial(5)) # 3. Importar con alias (apodo) import datetime as dt hoy = dt.date.today() print("Hoy:", hoy) # Bono: combinar from random import randint print("Dado:", randint(1, 6))

Archivos CSV

El formato universal de datos tabulares

¿Qué Es un Archivo CSV?

CSV = Comma Separated Values

Un archivo de texto plano donde cada línea es una fila y las columnas se separan por comas. Es el formato más universal para intercambiar datos tabulares.
