Múltiples Condiciones: & | ~

import pandas as pd df = pd.read_csv("titanic.csv") # & = Y (ambas condiciones verdaderas) mujeres_1ra = df[(df["Sex"]=="female") & (df["Pclass"]==1)] print(f"Mujeres en 1ra clase: {len(mujeres_1ra)}") # | = O (al menos una verdadera) ninos_o_1ra = df[(df["Age"] < 12) | (df["Pclass"]==1)] print(f"Niños O 1ra clase: {len(ninos_o_1ra)}") # ~ = NO (invierte la condición) no_southampton = df[~(df["Embarked"]=="S")] print(f"NO embarcaron en S: {len(no_southampton)}") # Combinación compleja analisis = df[(df["Pclass"]==3) & (df["Survived"]==1)] print(f"\nSupervivientes 3ra clase: {len(analisis)}") print(f"Tasa: {len(analisis)/len(df[df['Pclass']==3])*100:.1f}%")

Error común

Cada condición debe ir entre paréntesis. Sin paréntesis, Python aplica los operadores en el orden incorrecto.

# ✗ Error (no paréntesis)
df[df["Sex"]=="female" & df["Pclass"]==1]

# ✓ Correcto
df[(df["Sex"]=="female") & (df["Pclass"]==1)]

Filtros Especiales: isin() y between()

import pandas as pd df = pd.read_csv("titanic.csv") # isin() — como "IN" de SQL puertos = ["C", "Q"] no_southampton = df[df["Embarked"].isin(puertos)] print(f"Embarcaron en C o Q: {len(no_southampton)}") # between() — rango de valores adultos_medios = df[df["Age"].between(30, 50)] print(f"\nEdad 30-50: {len(adultos_medios)} pasajeros") print(adultos_medios[["Age","Sex","Survived"]].head(5)) # Negación con isin solo_southampton = df[~df["Embarked"].isin(["C","Q"])] print(f"\nSolo Southampton: {len(solo_southampton)}")
Cuándo usar cada uno:

isin(lista)

Cuando tienes múltiples valores válidos para una columna. Evita cadenas de |.

between(a, b)

Para rangos numéricos. Equivalente a (df["col"] >= a) & (df["col"] <= b) — pero más limpio.

Transformar Datos

Crear columnas derivadas con operaciones vectoriales y apply

Los datos crudos casi nunca vienen listos. Transformar es donde agregas inteligencia al dataset.

Crear Columnas Nuevas

import pandas as pd df = pd.read_csv("titanic.csv") # 1. Operación directa (vectorizada — muy rápida) df["Fare_doble"] = df["Fare"] * 2 # 2. Condición booleana como columna df["Es_mayor"] = df["Age"] >= 18 # 3. Categoría basada en condición df["Rango_edad"] = "Desconocido" df.loc[df["Age"] < 12, "Rango_edad"] = "Niño" df.loc[(df["Age"] >= 12) & (df["Age"] < 18), "Rango_edad"] = "Adolescente" df.loc[df["Age"] >= 18, "Rango_edad"] = "Adulto" print(df[["Age","Es_mayor","Rango_edad","Fare","Fare_doble"]].head(10))

  • df.loc[condición, "col"] = valor — asignación condicional segura.

apply() — Aplica una Función a Cada Valor

La idea en una frase

df["col"].apply(mi_funcion) toma cada valor de la columna, lo pasa por tu función, y devuelve una columna nueva con los resultados.

import pandas as pd df = pd.read_csv("titanic.csv") # Paso 1: defines una función normal def etiqueta_clase(numero): if numero == 1: return "1ra Clase" elif numero == 2: return "2da Clase" else: return "3ra Clase" # Paso 2: apply la pasa por cada valor de la columna df["Clase_texto"] = df["Pclass"].apply(etiqueta_clase) print(df[["Pclass", "Clase_texto"]].head(10))
¿Qué hace apply paso a paso? 
# Sin apply, harías esto:
resultados = []
for n in df["Pclass"]:
    resultados.append(etiqueta_clase(n))
df["Clase_texto"] = resultados

# apply() hace exactamente eso,
# pero en una sola línea:
df["Clase_texto"] = df["Pclass"].apply(
    etiqueta_clase
)

Cuándo usar apply

Cuando necesitas lógica con if/elif que no cabe en una sola expresión. Ej: clasificar clientes como "Premium" / "Regular" / "Nuevo".

groupby() + agg() — Varias Métricas a la Vez

¿Qué pasa si quieres calcular varias cosas al mismo tiempo? Usas .agg().

df.groupby("Pclass").agg(
    total    = ("PassengerId", "count"),
    tarifa_prom = ("Fare",         "mean"),
)
    ↑ nombre que tú eliges ↑ columna del df ↑ qué calcular
import pandas as pd df = pd.read_csv("titanic.csv") resumen = df.groupby("Pclass").agg( total = ("PassengerId", "count"), sobreviven = ("Survived", "sum"), tarifa_prom = ("Fare", "mean"), ).round(1) print(resumen)

La ventaja de .agg()

En lugar de hacer tres groupby separados y luego unirlos, .agg() los calcula todos de una vez y devuelve una tabla limpia.

El nombre es tuyo

total, sobreviven, tarifa_prom — tú decides cómo se llama cada columna del resultado.

¿Cómo Limpiar los NaN? — Compara las Estrategias Interactivo

Columna Age del dataset — 6 valores NaN de 30. Haz clic en cada estrategia para ver el efecto en la distribución y el promedio.

¿Cuándo usar cada una?

dropna()

Cuando los NaN son pocos (menos del 5%) y perder esas filas no importa.

fillna(mediana)

Para números. La mediana no se distorsiona por valores extremos. Primera opción para columnas numéricas.

fillna("valor")

Para columnas de texto o cuando hay un valor lógicamente correcto: el más frecuente, "Desconocido", etc.

fillna(0)

Casi nunca para datos reales. El 0 tiene significado propio — no es lo mismo que "dato faltante".

import pandas as pd df = pd.read_csv("titanic.csv") print(f"NaN en Age: {df['Age'].isna().sum()}") print(f"NaN en Embarked: {df['Embarked'].isna().sum()}") # Numérica → fillna con mediana mediana = df["Age"].median() df["Age"] = df["Age"].fillna(mediana) print(f"\nAge después: {df['Age'].isna().sum()} NaN (mediana={mediana})") # Categórica → fillna con valor lógico df["Embarked"] = df["Embarked"].fillna("S") # S es el más común print(f"Embarked después: {df['Embarked'].isna().sum()} NaN") # dropna → eliminar filas donde Fare sea NaN df2 = df.dropna(subset=["Fare"]) print(f"\ndropna(Fare): {len(df)} → {len(df2)} filas")
